> Preferred formats and patterns for organizing information across different business communication types.

## Core Communication Structures

### 1. Client Email (Professional Mode)

**Pattern**:
```
Subject: [Specific, clear subject with context]

Hi [Name],

[Opening: Connect to previous conversation or provide context]

[Body: Main point/recommendation/update - 2-3 focused paragraphs]
- Use bullets for multiple items or action steps
- Lead with most important information
- Keep each paragraph focused on one idea

[Closing: Clear next steps or call to action]

Best regards,
Ash

---
[Optional: Signature block for formal contexts]
```

**Best For**: Standard client communication, updates, recommendations

**Length**: 150-300 words typical (3-5 paragraphs)

**Key Elements**:
- Subject line sets clear expectation
- Opening connects to context
- Body is scannable (bullets, short paragraphs)
- Closing has clear next steps
- Professional but approachable tone

### 2. Meeting Summary

**Pattern**:
```
# Meeting Summary: [Clear Topic]

**Date**: [Date]
**Participants**: [Names and roles]
**Purpose**: [One-sentence meeting objective]

## Key Discussion Points
- [Point 1: Brief summary]
- [Point 2: Brief summary]
- [Point 3: Brief summary]

## Decisions Made
1. [Decision with brief rationale]
2. [Decision with owner if applicable]

## Action Items
- [ ] [Action item] - Owner: [Name] - Due: [Date]
- [ ] [Action item] - Owner: [Name] - Due: [Date]
- [ ] [Action item] - Owner: [Name] - Due: [Date]

## Next Steps
[What happens next / next meeting date]

## Open Questions
- [Question 1]
- [Question 2]
```

**Best For**: Post-meeting documentation, client meetings, internal meetings

**Length**: 200-500 words typical

**Key Elements**:
- Clear header information
- Structured sections (Discussion, Decisions, Actions)
- Checkbox action items with owners and dates
- Scannable format
- Next steps clearly stated

### 5. Quick Status Update

**Pattern**:
```
Subject: [Project Name] - Week of [Date] Update

Hi [Name],

Quick update on [project]:

✅ **Completed This Week**
- [Item 1]
- [Item 2]

🚧 **In Progress**
- [Item 1] - [% complete or status]
- [Item 2] - [% complete or status]

📋 **Next Week**
- [Item 1]
- [Item 2]

⚠️ **Blockers/Issues** (if any)
- [Issue with impact]

Let me know if you have questions or need more detail on anything.

Best,
Ash
```

**Best For**: Weekly client updates, sprint summaries, project check-ins

**Length**: 100-200 words typical

**Key Elements**:
- Clear subject line with timeframe
- Visual markers (emoji or bullets) for scan ability
- Organized by status (done/doing/next)
- Highlights blockers proactively
- Brief and actionable

## Paragraph & Sentence Structure

### Email Paragraphs
**Length**: 2-4 sentences typical
**Why**: Scannable on mobile; easy to digest
**Exception**: First paragraph often just 1 sentence (hook/context)

### Document Paragraphs
**Length**: 3-5 sentences typical
**Why**: More detailed analysis requires development
**Structure**: Topic sentence + supporting details + transition

### Sentence Rhythm

**Email/Casual**:
```
[Short opener]. [Medium development]. [Short conclusion].
```

**Formal Documents**:
```
[Topic sentence]. [Supporting detail with context]. [Additional supporting detail]. [Concluding insight].
```

## List Structures

### When to Use Numbered Lists
- Sequential steps (do 1, then 2, then 3)
- Prioritized items (most important to least)
- Options analysis (Option 1, Option 2, Option 3)

### When to Use Bulleted Lists
- Unordered items of equal weight
- Features or attributes
- Quick points or observations
- Action items (with checkboxes)

### List Item Format

**Short List Items** (preferred):
```
- Clear, concise point
- Another clear point
- Third point
```

**Detailed List Items** (when needed):
```
1. **Bold headline or key point**
   Supporting context or elaboration that provides necessary detail

2. **Next headline**
   More supporting context
```

## Visual Formatting

### Bold Text
**Use for**:
- Section headers
- Key terms first mention
- Emphasis on critical points
- List item headlines

**Avoid for**:
- Entire sentences (use sparingly)
- General emphasis (choose one or two words max)

### Markdown Headers

```
# Document Title (H1)
## Major Section (H2)
### Subsection (H3)
```

**Pattern**: Use consistently for hierarchy
**Avoid**: Skipping levels (H1 → H3)

### Line Breaks & White Space

**Email**: Line break between every paragraph
**Documents**: Standard paragraph spacing
**Lists**: Extra space between major sections

**Why**: Scan ability, visual breathing room

### Emoji Usage

**Casual Mode**: ✅ Acceptable for tone/organization
- ✅ ⚠️ 💡 📋 🚧 (status markers)
- 👍 🎯 (emphasis)

**Professional Mode**: 🤔 Use sparingly, if at all
- Occasional emoji for warmth in established relationships
- Never in first contact

**Formal Mode**: ❌ Never

## Communication Length Guidelines

### By Type

| Communication Type | Target Length | Why |
|-------------------|---------------|-----|
| Quick Email | 50-150 words | Respect time, get response |
| Standard Email | 150-300 words | Enough detail, still scannable |
| Meeting Summary | 200-500 words | Capture key points completely |
| Status Update | 100-200 words | Quick overview |
| Proposal | 1000-2000 words | Comprehensive but focused |
| Assessment Report | 2000-4000 words | Thorough analysis |
| Internal Note | Varies | Whatever you need |

### Length Principles

1. **Shorter is better** - If you can say it in fewer words, do
2. **Complete over brief** - Don't sacrifice clarity for brevity
3. **Scannable over dense** - Use structure to enable quick reading
4. **Action-oriented** - Cut anything that doesn't drive action or decision

## Structure Principles

### Principle 1: Lead with the Point
**Rule**: Most important information first
**Why**: Readers may not finish; respect their time
**Example**: "I recommend Option A" then explain why, not build up to it

### Principle 2: Make it Scannable
**Rule**: Use headers, bullets, short paragraphs
**Why**: Busy clients need to process quickly
**Example**: Break 500-word email into 5 sections with headers

### Principle 3: One Idea Per Unit
**Rule**: Each paragraph/section has one main point
**Why**: Cognitive load management
**Example**: Don't mix status update with new request in same paragraph

### Principle 4: Clear Next Steps
**Rule**: Every communication ends with what happens next
**Why**: Enables action, prevents confusion
**Example**: "Next step: Review by Friday, discuss Monday's call"

### Principle 5: Match Structure to Mode
**Rule**: Casual = loose structure, Formal = tight structure
**Why**: Appropriate for context and relationship
**Example**: Internal note can be rough; proposal must be polished
