## Project Lifecycle

### 1. Planning Phase
**Location**: `planned/` directory
**Actions**:
- Define objective and success criteria
- Identify dependencies and resources
- Estimate timeline
- Assign owner

### 2. Execution Phase
**Location**: Move to `active/` directory
**Actions**:
- Daily updates in PM tool (Linear/Height)
- Weekly exports to `experiences/projects/[name]/sprint-updates/`
- Regular check-ins during standups
- Adjust as needed

### 3. Completion Phase
**Location**: Move to `completed/` directory
**Actions**:
- Run `/review` on deliverables
- Run `/learn` to extract patterns
- Update project file with outcomes
- Archive for future reference

---

## Integration with PM Tool

### Daily Execution
- Task-level tracking happens in PM tool (Linear or Height)
- Standup discussions reference PM tool tasks
- Sprint planning and burn-down in PM tool

### Weekly Brain Sync
- Export PM tool status update to markdown
- Save to `experiences/projects/[name]/sprint-updates/week-[X].md`
- Brain reads these during retrospectives

### Monthly Retrospective
- Review all active projects
- Extract learnings with `/learn`
- Update project status in memory
- Plan next month's priorities
